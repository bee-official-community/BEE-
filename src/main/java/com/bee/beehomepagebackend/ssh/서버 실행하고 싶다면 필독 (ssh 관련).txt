VPC 개념을 이용해 RDS를 연결했습니다.
왜 이렇게 했냐면 RDS에 직접 접근하면 비용이 나오기 때문임. ( Public IPv4 유료화때문에 )

그래서 어떻게 하는거냐구요.
AWS EC2를 SSH 서버로 사용하고, 내 컴퓨터를 SSH 클라이언트로 사용하면 됨.
그리고 EC2로 RDS에 간접적으로 접근하면 됩니다.
이걸 SSH 터널링이라고 함.

근데 이런식으로 설명하면 바로 AWS를 포기하고 싶어지기 때문에
더 쉽게 설명하자면
같은 VPC(가상 네트워크)에 EC2와 RDS가 있다는 것을 이용해
EC2를 통해 RDS에 간접적으로 접근하는 것이라고 이해하시면 됩니다.

근데 이런거 다 필요없고 그냥 서버 실행하는 방법이 궁금하다면 아래를 따라하십쇼.
OpenSSH나 PuTTY를 사용해서 SSH 연결을 해놓고 스프링부트 서버를 실행시키면 됩니다.
사실 서버 실행시키면 자동으로 SSH 터널링 할 수 있게 자동화해보려고 했는데
시간이 너무 오래 걸려서 그냥 버렸습니다.

PuTTY에 설정 저장해놓고 서버 실행시키기 전에 PuTTY로 SSH 터널링합시다.
자료는 Confluence에 올려둘테니 참고하십쇼.
